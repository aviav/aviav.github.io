HIGH-CONVICTION PATTERNS - CONDENSED OVERVIEW
================================================================================

WHAT THIS SHOWS:

This file presents high-conviction evidence from 17 books (935 historical
patterns) evaluating whether AI will favor capability-enhancement (distributed
power, individual empowerment) or maximum-leverage (centralized control).

These patterns were extracted from comprehensive analysis of works by:
Acemoglu, Blum, Dictators Handbook, Ferguson, Girard, Graeber, Han Feizi,
Kahneman, Lessig, Ostrom, Perez, Piketty, Schmitt, Scott, Tufekci, Weber,
and Yudkowsky.

Each pattern represents a well-documented historical precedent with strong
evidence. Patterns are classified based on whether they support or refute
the core claims of "Eight Billion People With Datacenters: Why This Time
Might Actually Be Different" (Part 3 article).

CORE CLAIMS BEING EVALUATED:

1. AI makes comprehension easier than information control
2. Individual capability enhancement outpaces centralized control mechanisms
3. Historical "this time is different" failures might not apply to AI
4. Architectural features of AI favor distributed capability over bottlenecks

CLASSIFICATION SYSTEM:

Stance Categories:
  SS = STRONGLY SUPPORTS    - Pattern provides strong evidence FOR Part 3 claims
  S+ = SUPPORTS + Strong    - Clear support with strong documentation
  N+ = NEUTRAL + Strong     - Well-documented but doesn't favor either side
  R+ = REFUTES + Strong     - Clear challenge with strong documentation
  SR = STRONGLY REFUTES     - Pattern provides strong evidence AGAINST Part 3

Percentages (Supports% / Neutral% / Refutes%):
  Supports = SS + S+ combined
  Neutral = N+ alone
  Refutes = R+ + SR combined

READING THE DATA:

Format: [book] SS - S+ - N+ - R+ - SR (Sup% / Neu% / Ref%)

Example: [tufekci] 2 - 7 - 5 - 45 - 1 (15% / 8% / 77%)
  - Tufekci contributes 60 total high-conviction patterns
  - 77% challenge Part 3 (mainly through REFUTES + Strong evidence)
  - Consistent adversarial position across themes

AGGREGATION METHODS:

Each theme shows TWO aggregate calculations:

1. UNWEIGHTED: Simple sum of all book contributions
   - Each pattern counts equally
   - Shows raw distribution within theme

2. WEIGHTED: Normalized by each book's total contribution
   - Accounts for books with broader vs narrower focus
   - Formula: book_patterns_in_theme Ã— (book_total / 935)
   - Adjusts for different book coverage

When unweighted and weighted aggregates match closely, the distribution
is robust regardless of which books contribute most patterns.
